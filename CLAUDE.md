# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

Package manager is **pnpm** (see `pnpm-lock.yaml`, `pnpm-workspace.yaml`).

```bash
pnpm dev                 # Next.js dev server (default port 3000)
pnpm build               # production build
pnpm start               # serve the production build
pnpm lint                # eslint (flat config, eslint-config-next core-web-vitals + typescript)
pnpm exec tsc --noEmit   # typecheck — the fastest correctness gate; run this after edits
```

There is no test framework in this repo. `tsc --noEmit` plus a visual check in the browser is the verification loop.

`pnpm lint` currently reports one pre-existing error — `react-hooks/set-state-in-effect` in `growth-chart.tsx`, where the reduced-motion branch sets state synchronously. Everything else is clean; don't let that one mask a new one.

Note `next lint` was removed in Next 16 — the script is plain `eslint`.

## What this is

A single-page marketing site for "Midnight Surface", built as a **pixel-faithful implementation of a Figma design** (`Frame 1`, a 1440 × 18113 export). Next.js 16 App Router, React 19, Tailwind CSS v4, TypeScript strict.

`app/page.tsx` composes the whole page from one section component each, in scroll order: `SiteHeader → Hero → TrustedBy → Services → WhyUs → OurWork → Process → Pricing → Testimonials → Faq → CtaBanner → SiteFooter`. There is no routing beyond `/` — nav and CTA links point at routes (`/contact`, `/work`, …) that do not exist yet.

**The Figma is the source of truth for layout and colour.** Colours in `app/globals.css`, type sizes, line heights, and container widths were sampled/measured off the export, not eyeballed, and the inline comments record the measurement. When changing a value that has such a comment, re-measure rather than nudging it by feel.

**Body copy is NOT from the Figma.** The design shipped `Lorem Ipsum` throughout (FAQ, services, Our Work, one testimonial); that has been replaced with real copy written to the studio's actual offering — UI/web design, WordPress and React/Next builds, full-stack with databases, AI integration — and its real published policies: the free draft, $0 upfront, no commitment, 5-7 day delivery, $399 / $699 pricing. Testimonials and Our Work projects are realistic **placeholders** (fictional names/companies), flagged as such in each file, to be swapped for genuine ones. Keep new copy consistent with those facts, and note the whole page bans em-dashes (see the taste skill) — use commas, periods or parentheses.

## Architecture

### Styling: Tailwind v4, tokens only in `@theme`

`app/globals.css` is the entire design system. It holds the `@theme` token block (`--color-teal`, `--color-cream`, `--color-ink`, `--radius-card`, `--radius-pill`, …), the `pin-short` custom variant, and the global utilities: `.marquee-track`, `.lift`, `.bg-dotgrid`, `.badge-sticker`. There is no `tailwind.config` file — v4 config lives in the CSS.

Two constraints that are easy to break:
- The font family is applied in a plain `html { }` rule, **not** inside `@theme`. `next/font/local` emits a hashed family name into `--font-inter-display` at runtime, and `@theme` is resolved at parse time, so referencing it there silently fails.
- `app/layout.tsx` declares only the five Inter Display weights actually used (400, 400 italic, 500, 600, 700). Using a new `font-*` utility means adding the matching `.ttf` to `app/fonts/` and declaring it there first, or the browser synthesises the weight.

`.lift` is the shared hover physics — `cubic-bezier(0.16, 1, 0.3, 1)`, transform only. Reuse it instead of writing per-component hover transitions so the page decelerates as one system. Hover shadows were deliberately stripped from the whole page (`.lift`, the hero/header/CTA buttons): the design keeps every surface flat, and a raise reads as elevation on its own. Do not reintroduce a `hover:shadow-*`.

### Motion: Lenis for scroll, Motion for entry, GSAP only for pin/scrub

The split is deliberate and documented in each primitive:

- **`components/ui/smooth-scroll.tsx`** (Lenis) — mounted once in `app/layout.tsx`. It eases the real window scroll position, so everything downstream (ScrollTrigger, `useScroll`, `whileInView`) inherits the smoothing without knowing Lenis exists. Three bits of wiring are load-bearing and explained in the file: `autoRaf: false` + `gsap.ticker` (one frame, not two loops), `lenis.on("scroll", ScrollTrigger.update)`, and `lagSmoothing(0)`. Lenis' stylesheet is inlined into `globals.css` — its `height: auto` on html/body is what gives it something to scroll.
- **`components/ui/reveal.tsx`** (Motion) — `Reveal`, `RevealGroup` + `RevealItem` (variant-driven stagger; children read the parent's variants so inserting a card cannot desync the sequence), and `CurtainReveal` (a brand-colour panel that collapses via `scaleY` from a top origin). These and the `text-reveal.tsx` primitives share a `useReveal` hook rather than a bare `whileInView`, and that is **load-bearing for reloads**: with scroll restoration (`SmoothScroll`) a reload lands mid-page, so every reveal above the landing point was never scrolled through and a plain `whileInView` left it stuck at opacity 0 — content that reads as missing the moment you scroll back up. `useReveal` adds `passed`: any element already fully above the viewport at mount is shown at once (off screen, so its entrance is never seen, and it has settled by the time you reach it). The `setState` sits inside a `requestAnimationFrame` callback, which is both correct — measure after paint — and what keeps it clear of `react-hooks/set-state-in-effect`.
- **`components/ui/text-reveal.tsx`** — `TextReveal` (per-word mask rise; takes a plain **string** so it can split it, and sets `aria-label` so the heading is announced as one phrase) and `MaskReveal` (single mask, for a line containing markup). The `pb-[0.14em] / mb-[-0.14em]` pair on the mask is what stops `overflow-hidden` clipping descenders.
- **`components/ui/horizontal-pan.tsx`** (GSAP + ScrollTrigger) — the only place GSAP is used, because it needs real pinning and scrubbing. Used by both Our Work and How It Works. Set up through `gsap.matchMedia("(min-width: 768px) and (prefers-reduced-motion: no-preference)")`, so the pin simply does not exist on phones and reverts cleanly on resize. `start: "top top"` and `end: "+=<track distance>"` are load-bearing, as is the `ScrollTrigger.refresh()` after images settle. The section is exactly `100dvh` while pinned, so its `py` is kept tight — every px there is a px the cards do not get. `trailClassName` sets the room after the last card; raise it (How It Works uses `md:pr-[38vw]`) when something has to react to reaching the centre of the screen, because at the default the track stops with the last card at the right edge.
- **`components/process-track.tsx`** (`ProcessCarousel`) — the How It Works steps, with exactly one lit at a time. It owns the `HorizontalPan` (rather than being slotted into one) because the active index feeds both the cards inside the panning track and the pagination dots below it, which do not pan — that shared state needs one client-side parent; the section's `SectionIntro` header crosses the boundary as JSX. The active card translates up across the header rule (the design's line-crossing move), and the dots render in `HorizontalPan`'s `footer` slot. The highlight is driven by **nearest-to-centre**, measured from `useScroll` + `useMotionValueEvent`, not by an IntersectionObserver band: cards sit on a 384px pitch with a 24px gap, so any band wider than that gap catches two neighbours and any band narrower drops to zero while a gap crosses the middle. The axis flips at `md`, matching the pan. The cards carry the Figma's exact type scale by default; the `pin-short` custom variant (globals.css) compresses them on pinned viewports shorter than ~1080px, where the design's 562px card cannot fit inside 100dvh.
- **`components/pricing-card.tsx`** (`PricingGrid`) — the one entrance on the page that is a sequence rather than a single reveal. The two cards **converge from opposite edges** (the section's comparison argument made physical, and a movement no other section uses), then unpack in reading order, with the feature rows cascading and each tick springing on like a stamp. **The section has exactly one observer**, in `PricingGrid`, and it drives everything through `animate` bound to a boolean rather than `whileInView`. That is load-bearing, not incidental: a `whileInView` per card meant two IntersectionObservers on two elements of different height crossing their thresholds at different scroll positions, and the pair visibly drifted apart. There is no stagger *between* the cards either — they are a mirrored pair and start on the same frame. The entrance sits on a wrapper with `.lift` on the article beneath it; they cannot share an element, because Motion's inline transform beats the class and silently kills the hover lift.
- **`components/popular-badge.tsx`** — three beats, strictly in sequence: the pill springs in, the word mask-reveals using the same device as the headings, and only then does the yellow sheet behind it begin rocking. **The sequencing is what makes it read as smooth** — the sheet's CSS loop originally started at page load, so a parent and its child were transforming against each other while the pill was still settling, which no amount of spring tuning fixes. The class is applied on `onAnimationComplete`; one setState, once. **There is no blur anywhere on this page**: depth is offset sheets of flat brand colour (the star card's stacked green sheets, the coral bar running off the card edge), so the attention loop is a hard-edged yellow sheet (`.badge-sticker`), not a glow — a blurred halo was tried and read as borrowed from a different site. Opacity is pulled off the spring onto a plain tween, since a spring on opacity has no physical meaning and its clamped overshoot shows up as a flicker. Rotation is owned by Motion alone, because Tailwind's `rotate-*` composes with rather than replaces the transform Motion writes.
- **`components/price-counter.tsx`** — counts the price down from an anchor to what you pay. The value is a MotionValue whose formatted string Motion writes straight into the text node, so the count costs **zero React renders**; a `useState` tick per frame would re-render the card ~70 times. `animate()` is imperative so it cannot join a variant tree; it takes `started` from `PricingGrid`'s single observer rather than opening one of its own, and a `delay` to sit inside the card's cascade. Screen readers get the final price as static text and never hear the count.
- **`components/ui/marquee.tsx`** — GSAP-driven infinite scroll (the one non-pin place GSAP is used, by explicit request). It **measures**: a ResizeObserver reads one set's width and enough copies are rendered to over-fill the container, then GSAP slides the track by one set width on an endless repeat with a wrap modifier. This replaced a pure-CSS two-copy `-50%` translate, which left a gap on any screen wider than a single copy — the strip appeared to snap back. Item spacing must still stay `padding-right` on each item, never flex `gap`. The observers set state from their callbacks (not the effect body), which is what keeps it clear of `react-hooks/set-state-in-effect`.
- **`components/testimonial-carousel.tsx`** — Success Stories, on the shadcn/Embla carousel. **The progress bar is the clock**: the carousel advances on the fill animation's `animationend`, and there is no autoplay timer running beside it. `embla-carousel-autoplay` was tried first with the CSS bar set to the same delay, and the two drifted — they measure different things, since the plugin schedules around the scroll *settling* while the bar restarts when the slide index changes, which is when that scroll *begins*. Matching the numbers cannot fix it; the offset is the transition itself. Driving one from the other makes them one event, and hover-pause and off-screen-pause then come free and stay exact, because pausing the fill pauses the thing that triggers the advance. Two consequences to keep: only the **current** slide's bar carries the handler (all three fill, and three handlers would advance three slides), and the bar restarts by remounting on a `key`, which is React's version of the class-removal-plus-reflow hack. Nothing runs until the section is 40% in view.
- **`parallax.tsx`**, **`scroll-progress.tsx`** — scroll-linked drift, and the top progress rule, which fades out over the last 6% of the page so it is gone by the time the screen-tall footer arrives. (`magnetic.tsx` was removed: the pointer-attraction on the CTAs did not fit the product.)

Every animated primitive checks `useReducedMotion()` (or a `prefers-reduced-motion` media query in CSS) and degrades to static. Keep that when adding motion.

**The motion system lives in `components/ui/motion.ts`** and is the single source for how the page moves. `EASE` (the one decelerate curve, `[0.16,1,0.3,1]`, mirrored in CSS as `ease-[cubic-bezier(0.16,1,0.3,1)]`), `VIEWPORT` (reveal thresholds — `edge` 0.1 for above-fold/clipped, `in` 0.3 the default, `committed` 0.6 for set pieces that must fully arrive), and `DURATION` (`detail` 0.5 / `content` 0.7 / `display` 0.9, larger elements travel longer). Import from here rather than redefining `EASE` locally or picking a fresh threshold by feel — the reveal primitives default to `VIEWPORT.in`, so most sections should pass no `amount` at all and inherit the shared rhythm; only override for a documented `edge`/`committed` reason. The section openers (`section-intro.tsx`) all fire at that one threshold, sequenced by DOM order and delay, so every section opens as the same gesture.

Animate transform and opacity only. For scroll position use `useScroll` + `useMotionValueEvent` (as `site-header.tsx` does), not a raw `scroll` listener. `useMediaQuery` (`components/ui/use-media-query.ts`) wraps `matchMedia` in `useSyncExternalStore` — use it rather than reading `matchMedia` into state from an effect, which the `react-hooks/set-state-in-effect` rule rejects.

### Server vs client components

Default is Server Component; section components stay server-side and import client primitives. Keep it that way — the `"use client"` files are the primitives plus `growth-chart` and `site-header`.

Icons come from `@phosphor-icons/react/ssr` (the `/ssr` entrypoint), which is what lets server sections render icons without a client boundary. Import from `/ssr` in server components.

### Shared components

- `components/section-intro.tsx` — the label / heading / body trio that opens nearly every section, and the one place the entrance choreography lives (pill → heading assembling word by word → body catching up). Sections pass only what differs: the measured `max-w`, the colour, `align`. **The body type ramp is set here**, so callers pass width and colour only.
- `components/section-label.tsx` — `SectionLabel` (the pill eyebrow, `teal` / `onTeal`) and `sectionHeadingClass`, exported as a class string rather than a component because `SectionIntro` renders the `h2` through `TextReveal`, which must own the element to split it.
- `components/ui/accordion.tsx` — headless single-open accordion owning state and ARIA; styling is entirely the caller's (used by `faq.tsx` on cream cards and `services.tsx` on teal rows, which look nothing alike). The panel opens by transitioning `grid-template-rows: 0fr → 1fr`, **not** to a measured pixel height — the old measured-height version silently clipped content whose height it had guessed wrong, and `min-h-0` on the inner div is what lets it close. The ResizeObserver now only feeds the optional `fixedHeight` reserve.
- `components/growth-chart.tsx` — hand-built SVG with a scripted beat order (frame in → curve draws → marker arrives → green fill rises). The green wash is gated behind arrival on purpose; do not make it fill progressively. Because the whole thing is one 932-unit viewBox, its type is scaled by `COMPACT_TYPE_SCALE` below `sm` — geometry stays fixed, only the labels grow, and the callout pill is sized from the same factor so the box and its text grow together.

### Responsive

The design was drawn at 1440. Breakpoints in use are Tailwind defaults; two are not obvious:

- **Nav collapses to a menu below `lg`, not `md`.** At 768 the five links plus the wordmark and the CTA do not fit on one line — they wrapped and ran through the logo. Same reason the service panel only puts its tag list beside the paragraph at `lg`.
- **`md` is where the design's desktop layout starts**: the Our Work pin, the 3-column bento (`md:h-[760px]`), and the pricing grid. Below `md` the bento columns stack, and cards whose illustration is absolutely positioned carry an explicit `min-h-*` that is dropped again at `md` — without a fixed column height, `flex-1` has nothing to divide and those cards collapse to their heading.

Sections use `px-6 py-16 sm:py-20 md:py-28` (the design's `py-28` only at `md`). Custom hover CSS in `globals.css` is wrapped in `@media (hover: hover)` so it cannot latch on touch; Tailwind's own `hover:` variant already does this.

### Assets

`public/design/*` are exports from the Figma frame. `public/logo.png` is the mark — note it sets "Midnight" in **black**, for the white header; the footer uses `footer-logo.png`, the same wordmark drawn light, because black all but disappears on the teal.

**shadcn is now partly installed** (`button`, `carousel`, `progress` + `lib/utils.ts`). Everything else under `components/ui/` is still hand-written, and the two kinds do not mix: the hand-written primitives compose class strings directly, the shadcn ones use `cn`. Three things about the generated code are deliberate and will come back if you re-run `shadcn add --overwrite`:
- **`button.tsx` is rebuilt on this project's tokens.** As generated it referenced `bg-primary`, `border-ring`, `--radius-md` and friends; shadcn's init never ran here, so `globals.css` defines none of them and the button rendered as unstyled text. Variants are `primary` / `surface` / `onTeal`, sizes `sm` / `default` / `lg` / `block`. `ButtonLabel` is the hover mask-swap.
- **`carousel.tsx` has `CarouselPrevious` / `CarouselNext` removed.** The design has no arrows, and those two were the only thing importing `button.tsx` and **`lucide-react`** — a second icon family in a project standardised on Phosphor.
- **Both `carousel.tsx` and `progress.tsx` were edited for the lint rules.** The generated carousel called `onSelect(api)` synchronously inside an effect, which `react-hooks/set-state-in-effect` rejects; it subscribes only now and takes its first value from Embla's `reInit`.

`pnpm-workspace.yaml` approves build scripts for `sharp` and `unrs-resolver`. Without it pnpm 10 exits non-zero on **every** command, which takes `pnpm dev`, `pnpm lint` and the shadcn CLI down with it.

### In-page navigation

The nav links are hash anchors (`#top`, `#services`, `#work`, `#pricing`, `#success-stories`), rendered as **plain `<a>`, not `next/link`** — Link intercepts a same-page hash and runs its own scroll, which fights Lenis. The click is handled by one delegated listener in `SmoothScroll`; Lenis' own `anchors` option did not intercept, so it is off. Two things there are load-bearing: the target is resolved to a **number** with an explicit duration (handing Lenis the element left it resolving both position and timing from defaults, and it silently did nothing), and `onComplete` re-measures once, because the pinned sections resize the document mid-flight and the position computed at click time can land hundreds of px out. `HEADER_OFFSET` (96) is kept in step with the `scroll-mt-24` on each anchored section, which is the fallback when Lenis is not running.

**Scroll restoration is done by hand in `SmoothScroll`, and must stay that way.** A pin adds a spacer roughly as tall as the track it scrubs, and that height only exists once ScrollTrigger has run in an effect, after hydration. The browser restores the old position before then, against the short unpinned document — so a reload from Pricing clamps to the shorter height and lands the reader in Our Work, which looks exactly like a redirect. The browser cannot win that race, because its restore happens before the height it needs is knowable. So the position is saved on `pagehide` (plus `visibilitychange`, for tabs discarded while backgrounded) into `sessionStorage` and replayed once the pins exist; `history.scrollRestoration = "manual"` only stops the browser attempting its own broken version first. `ScrollTrigger.clearScrollMemory()` drops GSAP's own saved positions, which reapply on refresh and fight the same fight. The replay runs exactly twice — on the next frame (the earliest the pins can exist, so the hero is never seen) and again at 800ms to correct for `HorizontalPan`'s post-image refresh — and the second pass stands down if the reader has already scrolled, so a correction can never yank them back. An explicit `#hash` outranks the saved position.

## Gotchas

- **Tailwind v4 compiles `rotate-*` to the standalone `rotate` property, not `transform`.** `transition-[transform]` will never see it and the element snaps round with no animation. Transition `rotate` (see the service row arrow). Same applies to `translate-*` and `scale-*`.
- **A `whileInView` trigger must not sit on an element its own mask clips.** IntersectionObserver clips the intersection rect against ancestor `overflow`, so a child translated fully outside its mask has zero visible area and can never fire — it cannot animate until it is visible and cannot be visible until it animates. Put the trigger on the mask and drive the child with variants; both `TextReveal` and `MaskReveal` do this.
- Several component files carry a UTF-8 BOM and contain typographic characters (em dashes, `—`). Edits have previously mojibaked these on Windows; if a dash renders as `â€”`, that is an encoding regression, not intended copy.
- `@/*` maps to the repo root, so imports read `@/components/...`.
- The repo vendors design skills under `.claude/skills/` and `.agents/skills/` (pinned in `skills-lock.json`). The visual work in this repo was produced under `design-taste-frontend`; its rules — no raw scroll listeners, no layout-triggering animation, no default AI-gradient aesthetic — are what the code comments refer to as "the taste skill".
